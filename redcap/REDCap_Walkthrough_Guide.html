Core REDCap Concepts

Projects & Arms

A Project is your study container. Arms represent different study groups (e.g., treatment vs. control). Each arm can have its own timeline of events.

Events (Longitudinal)

Events are time points in your study (e.g., Baseline, Month 6, Month 12). They create the longitudinal structure that tracks participants over time.

Instruments / Forms

Instruments are the data collection forms (e.g., Demographics, ADOS-2 Scores). Each instrument maps to a set of fields and can be assigned to specific events.

Records

Each participant is a Record identified by a unique ID (e.g., nano_id). Records span across events, creating one row per participant-event combination.

Data Access Groups (DAGs)

DAGs restrict which records users can see — essential for multi-site studies. Your NANO study uses DAGs to separate sites.

The API

REDCap's REST API lets you programmatically export/import records, metadata, files, and reports using a simple token-based authentication.

Pull Implementation

Python
import redcap
import pandas as pd
from dotenv import load_dotenv
import os

load_dotenv()

# 1. Connect to REDCap
project = redcap.Project(
    os.environ["REDCAP_API_URL"],
    os.environ["REDCAP_API_TOKEN"]
)

# 2. Pull ALL records (flat format)
df = project.export_records(
    format_type="df",
    df_kwargs={"dtype": str}
)

# 3. Pull SPECIFIC event only
df_m12 = project.export_records(
    format_type="df",
    events=["month_12_arm_1"]
)

# 4. Pull SPECIFIC fields only
df_ados = project.export_records(
    format_type="df",
    fields=["record_id", "ados2_css_total"]
)

# 5. Save as Parquet
df.to_parquet("redcap_export.parquet", index=False)
Why Parquet? Parquet preserves types, compresses 5-10x smaller than CSV, and loads 10x faster. Your NANO pull script exports to Parquet by default.

bash
# Export all records as JSON
curl -X POST "https://redcap.institution.edu/api/" \
  -d "token=YOUR_TOKEN" \
  -d "content=record" \
  -d "format=json" \
  -d "type=flat"

# Export data dictionary (metadata)
curl -X POST "https://redcap.institution.edu/api/" \
  -d "token=YOUR_TOKEN" \
  -d "content=metadata" \
  -d "format=json"
Raw API: All REDCap API calls are POST requests to a single endpoint. The content parameter determines what you're requesting.

Push Code Example

Python
import redcap, pandas as pd

# 1. Load computed scores
df = pd.read_parquet("computed_scores.parquet")

# 2. Select only fields to push
push_cols = ["record_id", "redcap_event_name",
             "bayley4_cog_composite", "qc_flag"]
df_push = df[push_cols]

# 3. Convert to records and push
records = df_push.to_dict(orient="records")
project = redcap.Project(api_url, api_token)
response = project.import_records(
    records,
    overwrite="normal",  # only fill blanks
    return_format_type="json"
)
print(f"Pushed {response['count']} records")
Safety First: Always use --dry-run first! Your NANO push script supports python redcap_push.py --input scores.parquet --dry-run
