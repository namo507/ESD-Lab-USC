# HIPAA Compliance Checklist — NANO Study

## Encryption At Rest

| Asset | Encryption Method | Responsible Party |
|-------|------------------|-------------------|
| USC Secure Research Server | AES-256 (SC Research Computing managed) | IT / PI |
| Researcher laptops (macOS) | FileVault 2 enabled | Each researcher |
| Researcher laptops (Windows) | BitLocker enabled | Each researcher |
| USB drives (prohibited for PHI) | N/A — USB transfer of PHI is prohibited | All |
| REDCap database | TDE via USC REDCap instance | USC IT |

## Encryption In Transit

| Connection | Protocol | Notes |
|-----------|----------|-------|
| REDCap API calls | HTTPS / TLS 1.2+ | Enforced by USC REDCap server |
| Secure server access | SSH / SFTP | Key-based auth required |
| Email notifications | SMTP TLS (STARTTLS) | No PHI in email body |
| GitHub (code only) | HTTPS / SSH | No data in repo |

## Access Controls

- **REDCap user roles**: Data entry users cannot export; PI/analyst roles have export permission
- **GitHub**: Repository is private; team members added by PI only
- **Secure drive**: Requires VPN + university NetID authentication
- **Principle of least privilege**: Each team member has minimum access needed for their role

## Audit Logging

- **Data access audit**: `logs/data_access_audit.log` — generated by `@audit_trail` decorator
- **REDCap audit trail**: Built-in REDCap logging of all record views, edits, exports
- **Git history**: All code changes tracked with author + timestamp
- **Pipeline logs**: `logs/pipeline.log` records record counts at each step (no PHI values)

## Minimum Necessary Standard

- Analysts working with de-identified datasets never receive access to PHI fields
- REDCap exports for analysis must use the de-identified export option
- Data shared with Co-Is uses hashed participant IDs and age-in-days (not dates)

## Breach Response Protocol

1. **Contain**: Immediately revoke access / disconnect affected system; contact PI within 1 hour
2. **Assess**: Determine what data was exposed, how many participants, and how exposure occurred
3. **Notify IRB**: Submit incident report to USC IRB within 24 hours (irb@sc.edu)
4. **Notify participants**: If required by HIPAA breach notification rule (>500 records or significant harm risk)
5. **Document**: Complete USC Privacy Office breach report form; update security procedures to prevent recurrence
